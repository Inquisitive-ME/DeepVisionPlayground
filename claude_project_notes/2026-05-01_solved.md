# Single-object task — solved

Date: 2026-05-01

The single-shape detection task is now essentially solved.

## Final numbers

`--task heatmap`, 405k-param `CenterHeatmapNet`, lr=1e-4, 1000 epochs:

|     metric | value           |
|-----------:|:----------------|
|     median |   **2.3 px**    |
|       mean | 4.0 px          |
|   accuracy | **99.7%** (3 classes) |
| pearson_cx | 0.997           |
| pearson_cy | 0.997           |

256x256 canvas → median error of 2.3 px is < 1% of the image diagonal,
and 99.7% classification is statistically indistinguishable from solved
on 1000 val images (3 mistakes out of 1000).

## Why the previous setup couldn't get there

The FC-regression head plateaued at ~16 px median because the encoder
has stride 16 — the deepest features are 16×16 cells covering the whole
256×256 image. A per-image regression that maps "the whole feature map"
to a single (cx, cy) pair simply has no way to resolve finer than one
cell, no matter how big the FC head is. The FC head can interpolate
between cells but the gradient signal averages out at the stride floor.

Three things had to change:

1. **Heatmap output instead of per-image regression.** Predict a 64×64
   heatmap (stride 4) whose argmax is the center cell. Gives a coarse
   localization at one-quarter the cost of stride-1.
2. **Sub-pixel offset head.** A separate (2,) prediction per cell that
   refines argmax to true sub-pixel position. Sigmoided so it can't
   decode off-canvas.
3. **Penalty-reduced focal loss for the heatmap.** Plain BCE or MSE on
   the dense Gaussian target lets background gradients flatten the
   whole heatmap to zero (background pixels outnumber the peak ~4000
   to 1 at stride 4). Focal weights down well-classified background
   so the peak's gradient survives.

I also switched from BN to GroupNorm because BN running stats were
silently drifting on this fresh-data-every-epoch task, making
eval-mode predictions diverge from the train-mode loss. With
GroupNorm, train and eval behave identically.

```
epoch    mean_px   median_px   accuracy
    1     171.9       182.4       0.32  (random)
   50       7.2         5.4       0.49
  100       6.0         4.5       0.56

  800       4.4         2.4       0.99
 1000       4.0         2.3       0.997
```

Localization converges fast (median already <10 px by epoch 50),
classification climbs slowly throughout and lands near 100% by
~epoch 800.

## What's still left to do

- Multi-object: this commit handles single-shape only. Multi would
  need NMS on the heatmap (top-K peaks) and a matching step to align
  the K predictions with N ground-truth centers. Doable but a real
  follow-up.
- True pixel-perfect localization: 2.3 px median is essentially
  "stride 4 + sub-pixel residual error." Going to <1 px median would
  require a stride-1 or stride-2 decoder, which is much more compute
  per inference but should be doable on this small task.
- The FC-regression code path isn't dead; it's still useful as a
  baseline. Both paths share the same data, metrics, and TensorBoard
  scaffolding.

## Throughput

3,750 images/sec on a 3090 with `--gpu-data`. 1000 epochs of 1000
images takes ~270 seconds — about 4.5 minutes for a full convergence
run.
